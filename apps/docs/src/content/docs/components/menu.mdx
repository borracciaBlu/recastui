By using these components, you can create a customizable, accessible menu system for your RecastUI library.

## Keyboard Navigation
|Key|	Action|
|---|---|
|Enter or Space|	When MenuButton receives focus, opens the menu and places focus on the first menu item.|
|ArrowDown|	When MenuButton receives focus, opens the menu and moves focus to the first menu item.|
|ArrowUp|	When MenuButton receives focus, opens the menu and moves focus to the last menu item.|
|Escape|	When the menu is open, closes the menu and sets focus to the MenuButton.|
|Tab|	no effect|
|Home|	When the menu is open, moves focus to the first item.|
|End|	When the menu is open, moves focus to the last item.|

## Accessibility
- The `MenuButton` component has the appropriate ARIA attributes: `aria-haspopup`, `aria-controls`, and `aria-expanded`.
- The `MenuList` component has the `role="menu"` attribute and is only rendered when the menu is open.
- The `MenuItem` components have the `role="menuitem"` attribute.
- Keyboard navigation is supported within the menu items: ArrowUp, ArrowDown, Home, End, and Escape keys are handled.
- The `MenuItem` components' tabIndex is set to 0 when focused and -1 otherwise.
- Clicking outside the `MenuList` component will close the menu.

Please note that this implementation is a starting point and may require further improvements to meet all the accessibility requirements for your specific application.
